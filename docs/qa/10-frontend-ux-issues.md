# DBFox 前端对话与 SQL 结果展示问题梳理

> 创建日期：2026-06-21  
> 更新日期：2026-06-24 (全部问题已解决)  
> 范围：对话页、SQL 执行结果展示、Chart 渲染、对话持久化、设置页 UI 优化

## 总体状态总结

截至 2026-06-24，本规划文档中记录的所有 6 项前端 UX 和排版显示问题均已**全部修复并顺利通过回归测试**。系统在多 SQL 结果展示、ECharts 绑定稳定性、会话流式更新、历史记录持久化以及排版视觉体验方面达到了生产级的高保真要求。

---

## 问题清单及解决状态

### 1. Chart 渲染异常 【已解决 / RESOLVED】

#### 现象
对话或 SQL 结果中生成图表时，Chart 渲染存在异常（尺寸异常、切换丢失、多 SQL 场景绑定错乱）。

#### 修复说明
* **ECharts 渲染优化**：实现了后端序列化指标优先的 ECharts 数据映射，自动提取并计算数据维度。
* **数据绑定与隔离**：引入了 SQL 物理 ID / 哈希指纹机制，多 SQL 场景下每个 Chart 均与对应的结果集唯一绑定，并在切换会话、重新打开历史记录后能够百分之百正确恢复渲染。
* **尺寸适配**：重构了 Chart 容器，实现了响应式宽度自适应与空状态提示，消除了尺寸计算导致的图表压缩变形。

---

### 2. 多 SQL 执行结果只保留最后一条 SQL / SQL Artifact 只显示最后结果 【已解决 / RESOLVED】

#### 现象
当 AI 回复中包含多条 SQL 时，前端结果展示或工件区域只保留了最后一条 SQL 的结果，之前的被覆盖。

#### 修复说明
* **后端状态修复**：修复了 `DBFoxAgentState` 中重复声明 `analysis_units` 导致追加式 reducer 实效的逻辑 Bug。
* **前端结构重构**：前端状态管理 (`workspaceStore`) 支持数组形式存储多 SQL 执行状态和结果集。SQL Artifact 区域按执行顺序生成多个折叠块，每条 SQL 及对应的表格、图表独立渲染，避免相互覆盖。

---

### 3. 对话消息覆盖问题：AI 最后一条消息覆盖用户消息 【已解决 / RESOLVED】

#### 现象
对话流式输出过程中，由于消息定位逻辑粗糙，AI 的最新消息片段可能会覆盖掉用户刚发送的消息。

#### 修复说明
* **独立 ID 寻址**：移除了通过 `messages[messages.length - 1]` 盲目更新消息内容的历史逻辑。
* **精细化更新**：在用户发送消息时和 AI 开始回复时，分别创建具有全局唯一 UUID 的 `role: 'user'` 和 `role: 'assistant'` 消息占位节点。流式响应（SSE）仅定位和追加到特定的 `assistant` 消息节点，彻底杜绝了消息覆盖和顺序错乱的问题。

---

### 4. 持久化对话记录缺少前端入口 / 展示不完整 【已解决 / RESOLVED】

#### 现象
后端已具备对话持久化，但前端缺少完整入口，用户无法方便地查看、恢复、管理历史对话。

#### 修复说明
* **侧边栏历史会话列表**：在前端 AppShell 侧边栏集成了“历史对话”入口与会话列表展示，包含会话标题和更新时间。
* **会话生命周期管理**：完整实现了新建会话、切换会话、重命名会话、删除会话的前后端同步交互。
* **完整状态恢复**：切换会话时，能够恢复历史会话中所有的消息、SQL Artifact 以及 Chart 配置，确保数据连续性。

---

### 5. 设置页面显示优化 【已解决 / RESOLVED】

#### 现象
设置页面布局拥挤、信息层级不清、表单控件对齐不一致、保存状态反馈不足。

#### 修复说明
* **卡片式/标签页分组**：将大模型 API 凭据、数据库连接、安全门限配置等按类型分组并以 Tab 切换。
* **状态反馈增强**：添加了明确的“保存中”、“保存成功”、“连接测试中”、“测试失败”等加载状态与气泡提示。
* **安全保护**：对 API Key 等敏感凭据默认执行掩码遮罩（`••••••`）并提供明文切换开关。

---

### 6. 对话页排版和显示优化 【已解决 / RESOLVED】

#### 现象
对话页整体排版在多轮消息、多 SQL、多 Chart 场景下显得杂乱，字号、间距与排版层级有待优化。

#### 修复说明
* **单轨居中阅读布局**：将对话聊天区域整体重构为精美的单轨居中阅读流设计（定义 `--conv-rail-width: 860px`），两边留白以提升长文本可读性。
* **无干扰执行跟踪**：将 Agent 内部冗长复杂的工具调用 Trace Panel 从主对话阅读区中剥离，移入可折叠/悬浮的侧边面板，确保用户视线专注于 AI 的文本回复和核心数据。
* **排版细节调优**：使用 Outfit/Inter 字体对 Markdown 标题、列表、SQL 代码块进行了层级、边距和行高调优，极大地提升了专业 BI 工具的视觉高级感。
